✅ 核心技术更新（v1.2）

1. eBPF 内存管理修复：UMEM 描述符循环复用  
- 问题：AF_XDP 数据路径中 UMEM 描述符耗尽，导致内存池枯竭  
- 方案：重写接收循环，增加帧回收逻辑，维护本地帧池  
- 结果：10Gbps 流量下连续运行 24 小时，内存占用稳定，零泄漏  

2. ST 语法校验升级：基于 Lark 的完整 AST 解析  
- 问题：原正则匹配无法支持复杂 IEC 61131-3 ST 语法  
- 方案：定义完整文法，生成抽象语法树（AST），精准捕获语法错误  
- 优势：支持变量声明、条件/循环等全语法，为语义分析奠基  

3. 因果模型动态化：在线 Mini-Batch K-Means 聚类  
- 问题：硬编码阈值无法适应设备运行模式变化  
- 方案：基于历史数据在线聚类，支持概念漂移，状态数自动确定  
- 优势：无需人工调参，自适应不同工业场景
