## 🎯 系统定位
面向工业物联网、数字孪生、智能制造场景的**国产化边缘操作系统**，聚焦「内核级网络优化+工业通信+国产硬件适配」，从概念验证升级为可部署原型，支持10Gbps高吞吐、毫秒级端到端延迟，满足工业现场高可靠、高实时性需求。

## ✅ 核心技术修正（v1.2）
### 1. eBPF内存管理硬伤修复：UMEM描述符循环复用
- 问题解决：修复AF_XDP数据路径中UMEM描述符耗尽、内存池枯竭问题
- 核心实现：重写接收循环，增加帧回收逻辑，维护本地帧池保证性能
- 验证结果：10Gbps流量下连续运行24小时，内存占用稳定，无泄漏

### 2. ST语法校验完备性升级：基于Lark的完整AST解析
- 问题解决：替换原正则/简单堆栈检查，支持复杂IEC 61131-3 ST语法
- 核心实现：定义完整ST文法文件，基于Lark生成抽象语法树（AST），捕获所有语法错误
- 优势：支持变量声明、条件/循环语句等全语法，后续可扩展语义分析

### 3. 因果模型状态映射动态化：在线Mini-Batch K-Means聚类
- 问题解决：替换硬编码阈值，实现状态划分自适应工业场景
- 核心实现：基于设备历史数据在线聚类，支持概念漂移，状态数可配置/自动确定
- 优势：适配不同设备运行模式，无需人工调整阈值

## 🚀 战略升维部署路线
### 1. 硬件在环（HIL）实战：国产RK3588工业网关部署
- 目标：eBPF用户态程序移植ARM64，端到端延迟<50μs
- 核心任务：交叉编译eBPF内核对象、适配GMAC网卡驱动、基于libbpf/libxdp ARM64版本开发
- 时间计划：2周内完成移植与实测

### 2. 数字孪生接口打通：OPC UA over TSN接入
- 目标：物理世界与数字世界毫秒级同步，对接Unity/Unreal引擎
- 核心实现：集成open62541 OPC UA SDK，支持IEEE 802.1AS TSN时间同步，发布eBPF捕获数据为OPC UA变量节点
- 时间计划：3周内完成原型集成

### 3. 自主可控根技术：Rust重写ST语法解析器
- 目标：移除Python第三方库依赖，实现全栈自主可控（无GPL依赖）
- 核心实现：基于Rust pest/nom编写解析器，生成C FFI接口，支持Python/ C++调用
- 时间计划：4周内完成原型开发
